# BEVFormer 改造清单（仅规划，不做实际修改）

## 总目标
- 将 BEV 空间覆盖从 360°（环视）改为 180°（前向半空间/扇区）
- 模态从（可见光 + LiDAR）调整为（可见光 + Radar + 远红外 FIR），并移除对 LiDAR 文件/变换的硬依赖

---

## 1) 360° → 180° BEV 空间

### 任务清单
- 明确定义“180°”的几何含义  
  - 常见做法：仅前向半平面（例如 x ∈ [0, Xmax]，y ∈ [-Y, Y]）或前向扇区（还需角度裁剪策略）
- 调整 BEV 覆盖范围与网格分辨率  
  - 更新 `point_cloud_range`（影响 BEV 实际宽高、参考点采样、bbox coder 过滤范围）
  - 视需求更新 `bev_h_ / bev_w_`（BEV token 数、pos embedding 尺寸）
- 同步更新检测头与后处理的范围约束  
  - 更新 `bbox_coder.post_center_range` 以匹配新的 BEV 覆盖
  - 更新 `train_cfg.pts.point_cloud_range` 与相关过滤逻辑（若使用）
- 校验/同步时序运动补偿中“旋转中心”  
  - `rotate_center` 默认与 200×200 BEV 对齐，改变 BEV 尺寸后需要同步
- 回归验证点：确保 BEV 网格长度 `grid_length` 与新 `pc_range` 一致（否则 shift/尺度会不对）

## 2) 去除 LiDAR 模态（含“移除 LiDAR 硬依赖”）

> 现状：配置里 `use_lidar=False`，但数据与几何仍强依赖 `lidar2img / lidar2cam` 与 `lidar_path`。

### 任务清单
- 数据层：让 camera-only / camera+FIR / camera+radar 训练不再需要 `lidar_path`
  - `get_data_info()` 当前无条件写入 `pts_filename=info['lidar_path']`，需要改为可选/按模态
- 几何层：用非 LiDAR 的中间坐标系重建投影链路
  - 当前 `lidar2img` 由 nuScenes 的 `sensor2lidar_*` 推导，若彻底移除 LiDAR，应改为 `ego2cam + cam_intrinsic`（或直接提供 `ego2img`）并在 `img_metas` 里保持模型端可用
- 管线层：清理/替换仍在使用 `lidar2cam/lidar2img` 的增强与收集逻辑
  - `CropResizeFlipImage` 会重算 `lidar2img = cam2img @ lidar2cam`
  - `CustomCollect3D` 默认 meta_keys 里包含 `lidar2img/lidar2cam/pts_filename`

### 核心代码位置

## 3) 新增远红外（FIR）编码器 + 引入 Radar 模态（最终：可见光 + Radar + FIR）

### 任务清单（建议按“数据→编码→融合→训练/推理”拆解）
- 数据与标定
  - 在 `data_infos`（pkl）中加入 FIR 相机的 `data_path`、内参、外参（与 ego 或参考坐标系的关系）
  - 定义 radar 的输入形态：点云 / 体素 / BEV 栅格 / 已提取特征（不同形态决定后续编码器形状）
- 数据管线
  - 扩展加载与归一化：为 FIR 增加加载、`Normalize/Pad/Collect` 的 key 与 meta（例如额外的 `fir2img` 或与可见光共用的投影键）
  - 扩展 `CustomCollect3D.keys/meta_keys`：把 FIR、radar 必要字段打包进 `img_metas` 与 batch
  - 现有 `datasets/pipelines/loading.py` 为空文件，可作为未来新增 loader 的落点之一（当前不做改动）
- 模型编码与融合（两条典型路径，二选一或组合）
  - 路径 A：把 FIR 当成“额外相机视角”并入 `img`（N 维变大）
    - 修改 `num_cams`、相机 embedding（cams_embeds）长度匹配新的 N
    - 确保每个视角都有对应投影矩阵（模型端投影采样可用）
  - 路径 B：为 FIR 单独建 backbone/neck，再与可见光特征融合（concat/加权/注意力）
    - 修改 detector 的 `extract_img_feat` 或新增 `extract_fir_feat`，并在 head/transformer 前融合
  - Radar：实现 radar encoder 输出（例如 `radar_bev`）并与 BEV token 融合
    - 代码里已有 `MM_BEVFormerLayer` 预留 `lidar_bev/radar_bev` 入参，但目前配置未接入，需要把 encoder layer type 切到该层并补齐数据流
- 训练/推理接口一致性
  - 保证 `forward_train/forward_test` 的输入结构与 `CustomCollect3D` 输出一致（可见光/FIR/radar 都能走通）
  - 补齐可视化/评估若依赖特定 key（按需）

## 最小修改面（实现时的“先后顺序”建议）
1. 先完成 180°：只改 `pc_range + bev_h/w + rotate_center/post_center_range` 并跑通训练/推理
2. 再“彻底去 LiDAR 硬依赖”：把 `lidar2img` 生产链路改为 ego-based，保证 camera-only 可独立运行
3. 最后加 FIR 与 radar：先 FIR（更接近 camera），再 radar（需要新 encoder 与数据格式定义）
